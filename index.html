📝 게시 — Blogger 초안 생성(API)

작성한 글 HTML 파일을 Blogger 초안으로 바로 올립니다(publish-post). 기본은 초안이며 미리보기 확인 후 사람이 발행합니다(라이브는 --publish).

· 명령(PC): python studio/blogger.py publish-post --code <코드> --file posts/<글_HTML.txt> → 제목·라벨은 DB 코드에서 자동, 초안 생성.
· 먼저 확인: --dry-run을 붙이면 제목·라벨·본문길이만 확인(게시 안 함). 확인 후 빼고 실행.
· 라이브 발행은 --publish 명시(미지정 시 항상 초안). 초안은 Blogger에서 미리보기 후 게시.

· 가장 빠름: PC에서 C:\lifeinfo\커밋배포.bat 더블클릭 → 메시지 입력(엔터=기본) → 커밋+배포 한 방.

🟢 네이버 발행 — 브라우저 자동화

네이버 글쓰기 API는 2020년 종료돼 자동게시는 불가합니다. 대신 ①PC에서 네이버 글쓰기 패키지를 만들고(붙여넣기용) ②Claude+Chrome으로 글쓰기 화면을 조작합니다. 게시는 항상 승인 후.

· 패키지 생성(PC): python -m studio.pipeline naver-draft --code <코드> --file <네이버용 변형본> --open → 제목·변형본·관련글·태그가 클립보드에 복사되고 글쓰기 창이 열립니다. 붙여넣기 한 번.
· 변형본이 없으면 먼저 위 ③ OSMU 변환으로 네이버용을 만든 뒤 --file로 넣으세요(구글 원문 그대로 금지=저품질).
· 게시 후 URL: naver-link --code <코드> --url <URL> → 콘솔 연계도구 교차링크에 입력.

🛡️ 댓글 대응 — 검증·답변 초안

부정견해·사실오류 댓글을 웹·유튜브로 정밀검증해 출처 단 답변 초안과 메일을 만듭니다. 게시는 항상 승인 후(자동 반박 금지).

복사된 프롬프트를 Claude Code(데스크톱 앱)에 붙여넣으면 → 최근 댓글 점검 → 검증 리포트 + 답변 초안 2종 → Gmail 초안으로 받아봅니다. 댓글 게시·삭제는 건별 확인 후 진행.

🔍 색인 — 서치콘솔 색인 점검·요청

구글 서치콘솔에서 신규글이 색인됐는지 확인하고, 안 된 글은 색인 생성을 요청합니다. 색인 요청은 공식 API가 없어(브라우저 1클릭이 한계) Claude+Chrome으로 진행합니다.

🔎 서치콘솔 URL 검사 열기

· 색인상태 자동 표시(API): PC에서 python studio/searchconsole.py inspect 실행 → index_status.json 생성 → deploy 하면 모니터링 탭 '🔍 색인 현황'에 글별 ✓/✗ 표시. (최초 1회 searchconsole.py auth 동의 + GCP에서 'Search Console API' 사용설정)
· Claude에게 맡기기: 복사된 프롬프트를 Claude Code에 붙여넣으면 → DB 최근글을 GSC URL 검사로 점검 → '리디렉션 오류'는 실시간 테스트로 과거데이터(stale) 여부 판별 → 미색인 글 색인 요청(하루 ~10건).
· 일괄 경로: sitemap.xml(65편 전체)이 기본 색인 경로 — 개별 URL 요청은 신규글 가속용 보조입니다. sitemap이 '가져올 수 없음'이면 재제출.
· 한계: 색인 '요청'은 자동 API 불가(JobPosting/Broadcast 전용) → 사람/Claude 브라우저 1클릭. 결과는 며칠 뒤 반영.
