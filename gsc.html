📌 색인요청 사용법 (순서대로)
⚠️ 핵심 — 클린 URL은 모바일 302(?m=1) 때문에 "리디렉션 오류"로 색인 안 됨. 반드시 끝에 ?m=1 붙인 주소로 색인 요청해야 합니다(GSC가 색인 가능으로 확인함). [GSC 검사] 버튼이 ?m=1 주소를 자동 복사해 줍니다.
  1. [미요청만] 필터로 아직 요청 안 한 글만 봅니다.
  2. 각 줄 [GSC 검사(?m=1)] 클릭 → GSC 열림 + ?m=1 주소 자동 복사 → GSC 상단 검사창에 붙여넣기(Ctrl+V)+Enter
  3. 실제 URL 테스트 → "URL을 Google에 등록할 수 있음"(초록) 확인 → 파란 ‘색인 생성 요청’ 클릭 → “색인 생성을 요청했습니다” 팝업 확인.
  4. 콘솔로 돌아와 그 줄 ‘완료’를 직접 체크 — 실제로 요청한 글만 체크.
  5. 하루 약 10건 한도가 있어 초과 시 "할당량 초과"가 뜹니다 → 나눠서 진행.
  6. 실제 색인 여부(✅색인/⏳발견/⚠️리다이렉트/⛔미발견)는 며칠 뒤 PC에서 searchconsole.py inspect → deploy 하면 색인상태 칸에 갱신됩니다.
※ 완료 체크는 자동이 아니라 직접 합니다(구글은 요청 여부를 외부에 안 알려줌). 예전에 조회만 하고 체크된 게 있으면 아래 [체크 초기화]로 지우고 다시 정확히 관리하세요.
❓ ‘미발견 / 리다이렉트’는 왜 뜨고, 어떻게 하나요?
⛔ 미발견(URL unknown) = 구글이 아직 크롤을 안 해서 모른다는 뜻(요청을 눌렀어도 처리 전엔 이렇게 보입니다). 신규 블로그는 크롤이 느립니다.
⚠️ 리다이렉트(Redirect error) = 구글은 모바일 우선으로 크롤하는데, 블로그스팟은 모바일 접속 시 ?m=1로 302 이동합니다(그 페이지의 대표주소canonical는 깨끗한 URL). 블로그스팟 기본 동작이라 구글이 대개 canonical로 통합하며, 시간이 지나면 해소되는 경우가 많습니다.
✅ 실전 해법(GSC 검증됨) = 색인 요청은 반드시 ?m=1 붙인 주소로! 클린 URL은 302로 실패하지만 ?m=1은 "가져오기 성공·색인 생성 가능"으로 확인됨(canonical이 클린 URL이라 통합). [GSC 검사] 버튼이 자동으로 ?m=1을 복사합니다.
그 외 = 시간(신규 2~6주) · 트래픽/백링크(OSMU) · 꾸준한 발행.
※ ?m=1 색인이 나중에 빠질 수 있다는 보고도 있어 100% 영구보장은 아님. 다만 지금 시도할 검증된 방법. / ?m=1 테마 스크립트 강제 제거·커스텀 도메인은 효과 불확실(도메인도 같은 오류 보고)이라 권장하지 않음.
